# 11 — Standards map

Which external standards Fiduciary touches, and in which of three ways:

- **anchor**: the standard is ground truth inside the synthetic world;
  scenarios derive correct behavior from it.
- **map**: taxonomy controls carry informative references to it
  (standards/crosswalk.yaml), so evidence trails can be filed inside an
  adopter's own control structure.
- **adopt**: Fiduciary itself follows the practice the standard describes.

No conformance or certification is claimed for any entry. See
standards/README.md.

## AI governance frameworks

| Standard | Relation | Notes |
|---|---|---|
| NIST AI RMF 1.0 (NIST AI 100-1) | map | Crosswalk target for every wave-1 control |
| NIST GenAI Profile (NIST AI 600-1) | map | LLM-specific risk context |
| ISO/IEC 42001:2023 (AI management systems) | map | Benchmark evidence is the kind an AIMS operator needs for its own audits |
| ISO/IEC 42005:2025 (impact assessment) | map | Evidence chains resemble impact-assessment inputs |
| ISO/IEC 42006:2025 (requirements for AIMS auditors, published July 2025) | map | The audit ecosystem this benchmark feeds evidence into now has its own standard |
| ISO/IEC 23894:2023 (AI risk management) | map | Risk-level fields align with its vocabulary |
| ISO/IEC TR 24027 (bias) | map | Fairness controls |
| EU AI Act (Regulation (EU) 2024/1689) | anchor and map | The world excerpts it; Annex III 5(b) makes creditworthiness scenarios high-risk by definition |
| MAS FEAT + Veritas | map | The closest existing assessment methodology for AI in finance |
| OECD AI Principles | map | Family-level reference only |

Timing note (July 2026): the EU AI Act's high-risk obligations were
scheduled to apply from 2 August 2026. The Digital Omnibus proposal
(December 2025) defers them until harmonized standards, common
specifications, and guidance are available; CEN-CENELEC JTC 21 expects the
key standards (prEN 18228 risk management, prEN 18284 data quality) around
Q4 2026. Practical consequence for Fiduciary: organizations deploying
creditworthiness-adjacent AI need deployment evidence on exactly the
timeline this project is building for. Re-check this paragraph quarterly.

## Benchmark-governance precedents (not standards, but the bar to meet)

- NIST FRVT: sequestered test data and versioned public reports. The
  held-out scenario split (ADR 0008) is Fiduciary's path to the same
  property.
- MLCommons AILuminate and MLPerf: consortium governance, versioned
  releases, vendor-labeled submissions. GOVERNANCE.md starts the
  single-maintainer version of those rules.
- HELM: full transparency of prompts, predictions, and scoring. Fiduciary
  transcripts and judgements are stored per run for the same reason.
